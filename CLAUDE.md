## Project overview

Static marketing site for **עשור ושות׳ — עורכי דין** (Asor & Co. Attorneys at Law), a Jerusalem law firm specialising in urban renewal (פינוי ובינוי / תמ״א 38) and real-estate taxation. The site is published to `asor-law.com` via GitHub Pages (see `CNAME`, `.nojekyll`).

The repo contains **two parallel projects**:

1. **Marketing site** (root) — vanilla HTML/CSS/JS, multilingual (he / en / ar), generated from a single template.
2. **`worker/`** — Cloudflare Worker (Hono) that backs `/digital/` (the "האזור הדיגיטלי" e-commerce flow); integrates with SUMIT for payments.

`DESIGN.md` is the canonical design/content spec — refer to it for brand colors, typography, project list, and section structure.

## Build & dev commands

### Marketing site
```bash
node build.js          # regenerates he/index.html, en/index.html, ar/index.html
npx serve              # local preview (any static server works)
```

There is no test suite, no linter, and no CI. Deployment is `git push` to the default branch — GitHub Pages serves directly from the repo.

### Worker (`worker/`)
```bash
cd worker
npm install
npx wrangler dev       # local at http://localhost:8787
npm run deploy         # wrangler deploy
npm run tail           # live logs
```

One-time secret setup (not stored in repo):
```bash
npx wrangler secret put SUMIT_COMPANY_ID
npx wrangler secret put SUMIT_API_KEY
```

## Architecture: how the multilingual site is built

**Critical:** `he/`, `en/`, and `ar/` are **generated artifacts** — never hand-edit `{lang}/index.html`. Source files are:

- `template.html` — the single source HTML (~930 lines), uses `{{section.key}}` and `{{key}}` placeholders.
- `lang/he.json`, `lang/en.json`, `lang/ar.json` — translation tables; keys are grouped by section (`meta`, `header`, `hero`, `about`, `projects`, …). Hebrew is the default; `meta.dir` controls RTL/LTR per language.
- `build.js` — small Node script (no deps) that does string-replace of placeholders, plus toggles `{{lang_switcher_X_active}}` for the active-language `class="active"`. Missing keys log a warning and leave the placeholder in place; arrays are `JSON.stringify`'d (used for schema.org `knowsAbout`).

**Workflow when editing site content:** edit `template.html` and/or all three `lang/*.json` files, then run `node build.js`. Commit the regenerated `{he,en,ar}/index.html` along with the source changes.

`/index.html` at the repo root is a meta-refresh stub that redirects to `/he/`. The site is RTL-first; LTR styles only kick in for `en/` (where `meta.dir = "ltr"`).

### Cache-busting
`template.html` and `digital/*.html` reference CSS/JS with a version query string, e.g. `style.css?v=2026042902`. **Bump these `v=` values when changing `css/style.css` or `js/main.js`** so GitHub Pages / Cloudflare doesn't serve stale assets. The convention is `YYYYMMDD##`.

## Architecture: the digital products platform

`/digital/` is a separate static section (currently `<meta name="robots" content="noindex, nofollow">` — remove before public launch). It's not regenerated by `build.js`; the HTML is hand-written and Hebrew-only.

`digital/rental-agreement/form.js` is a 6-step questionnaire that:
1. Persists draft state to `localStorage` (`asor-rental-form-v1`).
2. On submit, will POST to the Worker → SUMIT → email DOCX. **The submit handler is currently a placeholder/`alert()`** (see TODO around line 380).

`worker/src/index.js` (Hono) exposes:
- `POST /api/rental/create-payment` — validates body, generates a `crypto.randomUUID()` order ID, calls SUMIT's `/billing/payments/beginredirect/`, returns `{ orderId, redirectUrl }` for client-side redirect.
- `GET /healthz`
- CORS allowlist comes from `wrangler.toml` `[vars] CORS_ORIGINS`.

`worker/src/sumit.js` wraps the SUMIT API. Note: SUMIT always returns HTTP 200 — success is `Status === 0`; anything else is an error.

The "persist `payload` keyed by `orderId` in KV/D1" step is **not yet implemented** (see TODO in `index.js`). Until it is, the flow trusts SUMIT's redirect parameters.

## Other things worth knowing

- **Contact form** uses Web3Forms (free service); access key is embedded in `js/main.js` and submissions go to `office@asor-law.com`. The form payload includes a metadata block (browser, UTM, referrer, time-on-page).
- **Accessibility panel** in `js/main.js` (~lines 400–610) implements Israeli-law-required a11y controls (font size, contrast, etc.) with prefs persisted to `localStorage`.
- **`categories/`** holds raw firm content (text + images) used as source material — not served by the site directly.
- **`מכתב גנרי לדיירים.pdf`** at the repo root is a public asset linked from the site.
- **Commit messages in this repo are written in Hebrew.** Match the existing style when committing (see `git log`).
- Phone number on site is `02-5000275` (office); WhatsApp/mobile is `054-6302880`. Don't change these without confirmation — they appear in many files including JSON-LD schema.
- `.env.example` at the root predates the Worker and lists older variable names; the live source of truth for Worker config is `worker/wrangler.toml` + `wrangler secret`.

## Don'ts

- Don't edit `he/index.html`, `en/index.html`, or `ar/index.html` directly — your changes will be overwritten by the next `node build.js`.
- Don't add a build framework, bundler, or package manager to the root. The site is intentionally framework-free.
- Don't commit `worker/.dev.vars`, `worker/.wrangler/`, or any `.env*` file (already gitignored).

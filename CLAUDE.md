# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository purpose

Marketing site + digital products platform for **Asor & Co. – Attorneys at Law** (עשור ושות׳), a Jerusalem law firm specializing in urban renewal (התחדשות עירונית) and real-estate taxation.

Production: `https://www.asor-law.com` (deployed via GitHub Pages — see `CNAME` and `.nojekyll`). The firm's brand identity, content sources, and project list live in `DESIGN.md` — read that first when changing copy, colors, or the projects grid.

## Build & run

There is no package manager at the repo root. The "build" is a single Node script.

```bash
node build.js              # Regenerates he/index.html, en/index.html, ar/index.html
npx serve .                # Local preview (root index.html redirects to /he/)
```

`build.js` reads `template.html` and the per-language JSON in `lang/{he,en,ar}.json`, expands `{{section.key}}` and `{{key}}` placeholders, and writes one `index.html` per language directory. **The generated files are committed to the repo** — GitHub Pages serves them directly. Always run `node build.js` and commit the regenerated `{he,en,ar}/index.html` after editing `template.html` or any `lang/*.json`.

Cache busting: `template.html`, `digital/index.html`, and `digital/rental-agreement/form.html` reference `css/style.css` and `js/main.js` with a `?v=YYYYMMDDNN` query string. Bump this version (one place per file) whenever shipping CSS/JS changes so GitHub Pages serves the new asset immediately.

### Worker (separate package under `worker/`)

```bash
cd worker
npm install
npx wrangler dev           # Local API at http://localhost:8787
npm run deploy             # wrangler deploy → https://asor-digital-api.<cf-subdomain>.workers.dev
npm run tail               # Stream production logs
```

Secrets (`SUMIT_COMPANY_ID`, `SUMIT_API_KEY`) are set via `npx wrangler secret put …`, **never** committed. Public vars live in `worker/wrangler.toml` under `[vars]`.

## Architecture

The repo contains three loosely-coupled pieces:

### 1. Marketing site (root) — static, multilingual, GitHub Pages

- `template.html` — single source of truth for the marketing page layout/markup. Do **not** hand-edit `he/index.html`, `en/index.html`, or `ar/index.html`; they are generated.
- `lang/{he,en,ar}.json` — all user-facing strings. Adding a `{{foo.bar}}` placeholder in `template.html` requires adding `foo.bar` to **all three** language JSONs (build prints `Missing translation` warnings otherwise but still emits the unresolved `{{…}}` literal).
- `css/style.css`, `js/main.js` — shared across all locales and across the `digital/` microsite. RTL is the default; LTR styling is applied via `[dir="ltr"]` selectors.
- `index.html` (root) — meta-refresh redirect to `/he/` plus hreflang alternates. The main entry for users is `/he/`.
- `he/privacy.html` is a **hand-written static page**, not generated by `build.js`. It is not in the build loop — edit it directly.
- The contact form in `js/main.js` POSTs to `https://api.web3forms.com/submit` with an embedded `access_key`. There is no backend for this form.

### 2. Digital products microsite — `digital/`

Separate landing page (`digital/index.html`) and product flows under `digital/<product>/`. Currently only `rental-agreement/` exists:

- `form.html` + `form.css` + `form.js` — 6-step wizard, RTL, with `localStorage` persistence keyed by `asor-rental-form-v1` (resumes mid-flow on reload).
- Submission flow (intended): form posts to the worker's `POST /api/rental/create-payment`, which calls SUMIT's `/billing/payments/beginredirect/` and returns a `redirectUrl`; the user is sent to SUMIT's hosted page; SUMIT redirects back to `RETURN_URL_BASE + /digital/rental-agreement/success.html?orderId=…`.
- `digital/index.html` is currently `<meta name="robots" content="noindex,nofollow">` — pre-launch. `form.html` is also noindex.

### 3. Backend Worker — `worker/` (Cloudflare Worker, Hono)

- `worker/src/index.js` — Hono app. CORS allowlist comes from `CORS_ORIGINS` env var (comma-separated). Routes: `POST /api/rental/create-payment`, `POST /api/agent/chat`, `GET /healthz`.
- `worker/src/sumit.js` — wraps SUMIT's `beginredirect` endpoint. **SUMIT always returns HTTP 200**; success is determined by `json.Status === 0`. Errors are surfaced via `UserErrorMessage` / `TechnicalErrorDetails`. The order's `ExternalReference` is our internal UUID, echoed back through SUMIT's redirect for verification.
- Currency code `1` = ILS. `Content-Language: he` header is required.
- Order persistence is **not yet wired** (see TODO in `index.js`); the worker currently trusts SUMIT's redirect parameters until KV/D1 storage is added.

### 4. AI intake agent — `worker/src/agent.js` + `digital/agent.js`

Stateless triage chat that maps a customer's free-text case description to a product from the catalog.

- **Catalog source of truth:** `worker/src/products.js` (`PRODUCTS` array — id, title, category, status, price, url, description). Mirror it whenever you add/remove a product card in `digital/index.html`.
- **Worker** (`agent.js`) calls Anthropic's `/v1/messages` with `claude-sonnet-4-6`, forces tool use on a single `respond_to_user` tool so the response is structured (reply, case_summary, recommended_product_id, needs_more_info, refer_to_office). Prompt caching is applied to system+tool blocks. Requires `ANTHROPIC_API_KEY` secret.
- **Frontend** (`digital/agent.js`) is stateless — sends the full `messages[]` history each turn. The chat container `#agentChat` reads its API base from `data-api` (empty → `http://localhost:8787` for `wrangler dev`; set to the deployed Worker URL before going live).
- `coming-soon` products are intentionally still recommended by the agent — the response copy explains that the office handles them via direct contact until the digital flow ships.

## Conventions

- **Hebrew is the primary language.** RTL is the default (`dir="rtl"` on `<html>`). Commit messages, source-content files in `categories/`, and most comments are in Hebrew. Don't rewrite Hebrew text into English unless asked.
- **No framework, no bundler, no transpilation** for the marketing site or the digital frontends. Vanilla HTML/CSS/JS only. Don't introduce build tooling without a clear reason.
- **Brand colors and typography come from `DESIGN.md`** — navy `#1B3A5C`, gold `#C5A55A`. Stay within these tokens.
- `categories/` holds firm-internal Hebrew text files (`01_פרטי_המשרד.txt`, etc.) used as **source content** when populating `lang/*.json` — they are reference material, not site assets.
- `templates/` (gitignored) contains firm-internal contract templates kept locally only — never commit content from there.
